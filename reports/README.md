# `reports/` — Per-intern weekly progress reports

Every intern keeps a folder here with one report per week.

## Folder convention

```
reports/
├── README.md             ← this file
├── _template/
│   └── week-N.md         ← copy this for each weekly entry
└── <github-handle>/
    ├── week-1.md
    ├── week-2.md
    ├── week-3.md
    ├── week-4.md
    ├── week-5.md
    ├── week-6.md
    └── final.md          ← Week-6 one-page contribution writeup
```

Use **your GitHub handle exactly** as the folder name (case-sensitive). Repo managers verify this matches your invite.

## Why this matters

This trail is what your **institute internship report** is built from at the end of the program. Skipping weeks = no audit trail = harder to defend your contribution. Each report should be authored by you, in your voice — not generated.

## Friday cadence

| Time (IST) | What |
|---|---|
| 5:00 PM | Demo Day — 60-second segment per intern (recorded) |
| 6:00 PM | Commit your `reports/<handle>/week-N.md` |
| 7:00 PM | Repo manager (Dhruv or Sohil) leaves a 1-line written feedback at the bottom of your entry — this becomes your mentor evaluation record |

## What "good" looks like

A good weekly report has:
- A specific named deliverable shipped (PR / commit / doc link)
- A short demo (Loom or screenshot)
- At least one metric (lines of code is not a metric; latency, accuracy, or % of corpus is)
- Honest blockers
- A specific plan for next week

A weak report has: "Worked on frontend. Will continue next week." Don't be that.
