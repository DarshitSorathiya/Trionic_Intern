# `docs/`

Long-form documents that don't belong inside a single package.

## What lives here

- [`TIMELINE.md`](./TIMELINE.md) — six-week milestone plan
- [`ARCHITECTURE.md`](./ARCHITECTURE.md) — system architecture + module map
- `RFC-*.md` — per-RFC design docs (one per significant subsystem)
- `runbooks/` — on-call and operational runbooks (DevOps owns)
- `CORPUS.md` — corpus priority list (Team D will author)

## Authoring rules

- One file per topic. Keep them short.
- Diagrams are welcome — ASCII or embedded images under `docs/img/`.
- RFCs follow the issue template at `.github/ISSUE_TEMPLATE/rfc.md`.
