# RFC: Citation Drawer & PageIndex Tree Path UX

## Author

Tirth Gondaliya

## Team

Frontend — Citation Drawer + Tree Path Links

## Status

Draft

---

# 1. Problem

Legal citations generated by the system contain hierarchical PageIndex paths such as:

IPC › Chapter XVI › Section 375 › Explanation 2

Users currently do not have a structured way to:

* inspect citation hierarchy
* preview referenced legal text
* navigate deep legal nodes
* keep citations visible while editing drafts

This RFC proposes a citation drawer UX for rendering and interacting with PageIndex citation trees inside the editor.

---

# 2. Goals

The proposed citation drawer should:

* Render hierarchical PageIndex tree paths clearly
* Support hover previews for quick inspection
* Allow click-to-pin behavior while editing
* Support expandable/collapsible citation trees
* Be keyboard accessible
* Minimize disruption to drafting workflow

---

# 3. Non-Goals

This RFC does not cover:

* Citation generation logic
* Backend retrieval APIs
* Editing citation node data
* PageIndex ingestion or indexing
* Legal validation logic

---

# 4. Proposed UI

## 4.1 Inline Citation Badge

Legal references inside the editor appear as compact citation badges.

Example:

[IPC §375]

Hovering the badge shows a small preview tooltip.

Clicking the badge opens the citation drawer.

---

## 4.2 Citation Drawer

The citation drawer appears on the right side of the editor.

The drawer contains:

* hierarchical PageIndex tree
* legal text preview
* node metadata
* snapshot/version information (future enhancement)

Example structure:

IPC
└── Chapter XVI
└── Section 375
└── Explanation 2

---

## 4.3 Expand / Collapse Behavior

Tree nodes support collapsible navigation.

Interactions:

* Click expand icon → expand node
* Click collapse icon → collapse node
* Active node remains highlighted

Collapsed nodes improve readability for deeply nested acts.

---

## 4.4 Hover Behavior

Hovering over a citation badge:

* shows short legal preview
* highlights referenced node
* does not pin drawer

Hover previews disappear when pointer exits.

---

## 4.5 Click-to-Pin Behavior

Clicking a citation:

* opens the citation drawer
* pins the selected citation
* keeps drawer open during editing

This helps users compare drafted text with source law.

---

# 5. Keyboard Navigation

The citation drawer should support keyboard accessibility.

| Key   | Action               |
| ----- | -------------------- |
| ↑ / ↓ | Navigate tree nodes  |
| →     | Expand node          |
| ←     | Collapse node        |
| Enter | Open/select citation |
| Esc   | Close drawer         |
| Tab   | Move focus           |

Focus state should always remain visible.

---

# 6. Accessibility

The citation tree should follow ARIA accessibility guidelines.

Proposed support:

* role="tree"
* role="treeitem"
* keyboard navigation
* focus indicators
* screen-reader friendly labels

Accessibility is important because legal drafting workflows may involve long reading sessions and power users.

---

# 7. Suggested Component Structure

Possible frontend components:

* CitationBadge
* CitationTooltip
* CitationDrawer
* CitationTree
* CitationNode
* CitationPreviewPanel

---

# 8. Suggested Implementation

Potential implementation using current frontend stack:

* Next.js App Router
* Tailwind CSS
* shadcn/ui Sheet component for drawer
* Collapsible component for tree expansion
* Tooltip component for hover preview

State management may use:

* React Context

---

# 9. Risks & Edge Cases

Potential concerns:

* Very deep tree nesting
* Long legal text overflow
* Small-screen responsiveness
* Multiple citations open simultaneously
* Performance with large legal trees

---

# 10. Future Improvements

Possible future enhancements:

* Search inside citation drawer
* Multiple citation compare mode
* Highlight referenced spans in editor
* Citation history navigation
* Snapshot version diff viewer
